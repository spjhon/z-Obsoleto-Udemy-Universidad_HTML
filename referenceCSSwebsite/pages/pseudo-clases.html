PSEUDO-CLASES

Son selectores que permiten aplicar estilos a un elemento en función de su estado o posición en relación a otro elemento. Se utilizan para seleccionar y aplicar estilos a elementos que no se pueden seleccionar con selectores de clase o ID, como por ejemplo el primer elemento de una lista, un elemento al que se ha hecho clic o un elemento que se encuentra en un estado de foco.

Selecciona elementos que están en un estado específico o que cumplen ciertas condiciones (por ejemplo, :hover seleccionará un elemento cuando el cursor se coloque sobre él).

:active

The active pseudo-class selector selects the element when selecting the element with the mouse (the mouse-click).

Tener en cuenta: LVHA-order: :link — :visited — :hover — :active.

:any-link

It matches every a or area element that has an href attribute. Thus, it matches all elements that match :link or :visited.

See the Pen Pseudo-Clase :any-link by Juan Camilo (@spjhon) on CodePen.

:autofill

The :autofill CSS pseudo-class matches when an input element has its value autofilled by the browser. The class stops matching if the user edits the field.

See the Pen Pseudo-Clase :autofill by Juan Camilo (@spjhon) on CodePen.

:checked

The :checked CSS pseudo-class selector represents any radio (<input type="radio">), checkbox (<input type="checkbox">), or option (<option> in a <select>) element that is checked or toggled to an on state.

See the Pen Pseudo-Clase :checked by Juan Camilo (@spjhon) on CodePen.

:default

The :default CSS pseudo-class selects form elements that are the default in a group of related elements.

See the Pen Pseudo-Clase :default by Juan Camilo (@spjhon) on CodePen.

:disable

The :disabled CSS pseudo-class represents any disabled element. An element is disabled if it can't be activated (selected, clicked on, typed into, etc.) or accept focus. The element also has an enabled state, in which it can be activated or accept focus.

See the Pen Pseudo-Clase :disable by Juan Camilo (@spjhon) on CodePen.

:empty

The :empty CSS pseudo-class represents any element that has no children. Children can be either element nodes or text (including whitespace). Comments, processing instructions, and CSS content do not affect whether an element is considered empty.

The :empty pseudo-class can be useful if you have little control over the HTML and want to hide empty elements, such as a WYSIWYG content editor. Here, an editor has added a stray, empty paragraph.

See the Pen Pseudo-Clase :empty by Juan Camilo (@spjhon) on CodePen.

:enabled

The :enabled CSS pseudo-class represents any enabled element. An element is enabled if it can be activated (selected, clicked on, typed into, etc.) or accept focus. The element also has a disabled state, in which it can't be activated or accept focus.

See the Pen Pseudo-Clase :enabled by Juan Camilo (@spjhon) on CodePen.

:first-child

The :first-child CSS pseudo-class represents the first element among a group of sibling elements.

See the Pen Pseudo-Clase :first-child by Juan Camilo (@spjhon) on CodePen.

:first-of-type

The :first-of-type CSS pseudo-class represents the first element of its type among a group of sibling elements.

See the Pen Pseudo-Clase :first-of-type by Juan Camilo (@spjhon) on CodePen.

:focus

The :focus CSS pseudo-class represents an element (such as a form input) that has received focus. It is generally triggered when the user clicks or taps on an element or selects it with the keyboard's Tab key.

See the Pen Pseudo-Clase :focus by Juan Camilo (@spjhon) on CodePen.

:focus-within

The :focus-within CSS pseudo-class matches an element if the element or any of its descendants are focused.

See the Pen Pseudo-Clase :focus-within by Juan Camilo (@spjhon) on CodePen.

:fullscreen

The :fullscreen CSS pseudo-class matches every element which is currently in fullscreen mode. If multiple elements have been put into fullscreen mode, this selects them all.

See the Pen Pseudo-Clase :fullscreen by Juan Camilo (@spjhon) on CodePen.

:has()

The functional :has() CSS pseudo-class represents an element if any of the relative selectors that are passed as an argument match at least one element when anchored against this element. This pseudo-class presents a way of selecting a parent element or a previous sibling element with respect to a reference element by taking a relative selector list as an argument.

See the Pen Pseudo-Clase :has() by Juan Camilo (@spjhon) on CodePen.

:hover

The :hover CSS pseudo-class matches when the user interacts with an element with a pointing device, but does not necessarily activate it. It is generally triggered when the user hovers over an element with the cursor (mouse pointer).

See the Pen Pseudo-Clase :hover by Juan Camilo (@spjhon) on CodePen.

:in-range

The :in-range CSS pseudo-class represents an input> element whose current value is within the range limits specified by the min and max attributes.

See the Pen Pseudo-Clase :in-range by Juan Camilo (@spjhon) on CodePen.

:indeterminate

The :indeterminate CSS pseudo-class represents any form element whose state is indeterminate, such as checkboxes which have their HTML indeterminate attribute set to true, radio buttons which are members of a group in which all radio buttons are unchecked, and indeterminate progress> elements.

See the Pen Pseudo-Clase :indeterminate by Juan Camilo (@spjhon) on CodePen.

:invalid

The :invalid CSS pseudo-class represents any form>, fieldset>, input> or other form> element whose contents fail to validate.

See the Pen Pseudo-Clase :invalid by Juan Camilo (@spjhon) on CodePen.

:is()

The :is() CSS pseudo-class function takes a selector list as its argument, and selects any element that can be selected by one of the selectors in that list. This is useful for writing large selectors in a more compact form.

See the Pen Pseudo-Clase :is() by Juan Camilo (@spjhon) on CodePen.

:lang()

The :lang() CSS pseudo-class matches elements based on the language they are determined to be in.

See the Pen Pseudo-Clase :lang() by Juan Camilo (@spjhon) on CodePen.

:last-child

The :last-child CSS pseudo-class represents the last element among a group of sibling elements.

In cases where there is only one element, that element counts as the first-child, only-child and last-child!

See the Pen Pseudo-Clase :last-child by Juan Camilo (@spjhon) on CodePen.

:last-of-type

The :last-of-type CSS pseudo-class represents the last element of its type among a group of sibling elements.

See the Pen Pseudo-Clase :last-of-type by Juan Camilo (@spjhon) on CodePen.

:link

The :link CSS pseudo-class represents an element that has not yet been visited. It matches every unvisited a> or area> element that has an href attribute.

See the Pen Pseudo-Clase :link by Juan Camilo (@spjhon) on CodePen.

:not()

The :not() CSS pseudo-class represents elements that do not match a list of selectors. Since it prevents specific items from being selected, it is known as the negation pseudo-class.

See the Pen Pseudo-Clase :not() by Juan Camilo (@spjhon) on CodePen.

:nth-child()

The :nth-child() CSS pseudo-class matches elements based on their position among a group of siblings.

See the Pen Pseudo-Clase :nth-child() by Juan Camilo (@spjhon) on CodePen.

:nth-last-child()

The :nth-last-child() CSS pseudo-class matches elements based on their position among a group of siblings, counting from the end.

See the Pen Pseudo-Clase :nth-col() by Juan Camilo (@spjhon) on CodePen.

:nth-last-of-type()

The :nth-last-of-type() CSS pseudo-class matches elements based on their position among siblings of the same type (tag name), counting from the end.

See the Pen Pseudo-Clase :nth-last-of-type() by Juan Camilo (@spjhon) on CodePen.

:nth-of-type()

The :nth-of-type() CSS pseudo-class matches elements based on their position among siblings of the same type (tag name).

See the Pen Pseudo-Clase :nth-of-type() by Juan Camilo (@spjhon) on CodePen.

:only-child

The :only-child CSS pseudo-class represents an element without any siblings. This is the same as :first-child:last-child or :nth-child(1):nth-last-child(1), but with a lower specificity.

See the Pen Pseudo-Clase :only-child by Juan Camilo (@spjhon) on CodePen.

:only-of-type

The :only-of-type CSS pseudo-class represents an element that has no siblings of the same type.

See the Pen Pseudo-Clase :only-of-type by Juan Camilo (@spjhon) on CodePen.

:optional

The :optional CSS pseudo-class represents any input>, select>, or textarea> element that does not have the required attribute set on it.

See the Pen Pseudo-Clase :optional by Juan Camilo (@spjhon) on CodePen.

:out-of-range

The :out-of-range CSS pseudo-class represents an input> element whose current value is outside the range limits specified by the min and max attributes.

See the Pen Pseudo-Clase :out-of-range by Juan Camilo (@spjhon) on CodePen.

:placeholder-show

The :placeholder-shown CSS pseudo-class represents any input> or textarea> element that is currently displaying placeholder text.

See the Pen Pseudo-Clase :placeholder-show by Juan Camilo (@spjhon) on CodePen.

:read-only

The :read-only CSS pseudo-class represents an element (such as input or textarea) that is not editable by the user. Select and style only if the input element is "readonly"

See the Pen Pseudo-Clase :read-only by Juan Camilo (@spjhon) on CodePen.

:read-write

Select and style only if the input element is not "readonly"

See the Pen Pseudo-Clase :read-write by Juan Camilo (@spjhon) on CodePen.

:required

The :required CSS pseudo-class represents any input>, select>, or textarea> element that has the required attribute set on it.

See the Pen Pseudo-Clase :required by Juan Camilo (@spjhon) on CodePen.

:root

The :root CSS pseudo-class matches the root element of a tree representing the document. In HTML, :root represents the html> element and is identical to the selector html, except that its specificity is higher.

See the Pen Pseudo-Clase :root by Juan Camilo (@spjhon) on CodePen.

:target

The :target CSS pseudo-class represents a unique element (the target element) with an id matching the URL's fragment.

See the Pen Pseudo-Clase :target by Juan Camilo (@spjhon) on CodePen.

:valid

The :valid CSS pseudo-class represents any input> or other form> element whose contents validate successfully. This allows to easily make valid fields adopt an appearance that helps the user confirm that their data is formatted properly.

See the Pen Pseudo-Clase :valid by Juan Camilo (@spjhon) on CodePen.

:visited

The :visited CSS pseudo-class applies once the link has been visited by the user. For privacy reasons, the styles that can be modified using this selector are very limited. The :visited pseudo-class applies only a> and area> elements that have an href attribute.

See the Pen Pseudo-Clase :visited by Juan Camilo (@spjhon) on CodePen.
